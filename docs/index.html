SPEC-DRIVEN DEVELOPMENT STARTER PACK

Launch spec-driven, multi-agent projects with safer defaults, clearer onboarding, reusable profiles, and a repo-first workflow contract.

View on GitHub See first release

Based on GitHub's spec-kit, then extended into a fuller starter pack for onboarding, workflow governance, profiles, add-ons, and project setup.

Flow

idea → spec → plan → build → repo ready

What changes here

  • repo-local workflow contract
  • profiles for common project shapes
  • add-ons for databases and orchestration
  • beginner-friendly generated setup

Why this exists

More structure than a toolkit, less weight than a platform.

The pack is designed for builders who like the spec-driven method but also need a more practical starting point for new repositories.

Common friction

New repos often start with unclear setup, inconsistent prompts, weak onboarding, and no shared workflow rules.

What this pack adds

It turns the method into a ready-to-use starter with guided project generation, governance, and portable defaults.

Who it helps

Solo builders, teams, and agent-heavy repos that want faster starts without losing clarity or control.

Core pieces

Built to stay useful after the first scaffold.

profiles

Reusable project shapes for APIs, data work, automation, and web stacks.

add-ons

Composable bundles for databases, orchestration, and common capability layers.

workflow contract

Generated prompts, docs, and manifest files stay aligned around a repo-first source of truth.

Quickstart

Get the pack locally in a few commands.

Clone

git clone https://github.com/williamceccon/spec-driven-development-starter-pack.git
cd spec-driven-development-starter-pack

Bootstrap

./scripts/bootstrap.ps1
./scripts/install-workflow-pack.ps1
./scripts/doctor.ps1

Generate

./scripts/new-project.ps1
# or use the shell scripts on macOS / Linux
